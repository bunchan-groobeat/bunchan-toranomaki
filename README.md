# ぶんちゃんの虎の巻（bunchan-toranomaki）

社長専用の「生きた虎の巻」。毎日の仕事の学びを1本ずつ足して育てる、鍵付きの教材サイト。

- 公開：GitHub Pages（noindex・nofollow）
- 器：`index.html` 1枚（完全同一オリジン・外部CDN依存ゼロ・外部送信なし）
- 配色：黒×白×赤（朱 `#C0392B`）のシンプル基調
- 鍵：アクセスキーのSHA-256をindex.htmlに埋め込み、クライアント側で照合（簡易鍵）

## ⭐ 学びを1本足す（最短導線）

1. `learnings/` に **次の連番の `.md` を1枚追加**（例：`004.md`）。
2. ファイル先頭にフロントマターを書く：
   ```
   ---
   title: タイトル
   date: 2026-07-21
   tags: タグ1, タグ2
   ---
   本文（Markdown）
   ```
3. `git add → commit → push`。**それだけで「今日の学び」の先頭に反映**されます。

- 番号は `001` から連番（`000` 埋めの3桁）。**新しい番号ほど先頭**に出ます。
- サイトは `learnings/001.md` から順に読み込み、**3つ連続で見つからなければ停止**します（少しの欠番は許容・大きな飛びは避ける）。
- 使える記法：見出し `#`〜`####`、段落、箇条書き `- `、**太字**、`インラインコード`、コードブロック、`[リンク](https://…)`。

## 🔑 鍵について（簡易鍵の限界）

- 鍵は「うっかり見られる」を防ぐ簡易ゲート。**強い秘匿は保証しません**。
- 公開リポのため、鍵を知らなくても `learnings/*.md` は直接読めます。
- ゆえに **秘密情報・顧客名・金額など機微な情報は載せない**運用で守ります。
- 将来 Firebase 認証等へ差し替え可能な構造（鍵照合を1箇所に閉じてある）。
- 鍵の変更：新しいキーのSHA-256を計算し、`index.html` の `KEYHASH` を差し替え→push。
