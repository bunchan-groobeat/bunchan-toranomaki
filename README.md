# ぶんちゃんの虎の巻（bunchan-toranomaki）v2

社長・ターミナルAI・スタッフが共通で使う「**現場の技術リファレンス**」。仕事前に一読／詰まったら引く、の両用。
模範＝師匠（ダブデザイン aizu-local.shop）の型：**工程番号カード × 担当タグ × 押すと開くプロンプト**。

- 公開：GitHub Pages（noindex・鍵ゲート）。鍵はアクセスキーのSHA-256を `index.html` に埋込・平文はリポ外。
- 器：`index.html`（完全同一オリジン・外部依存ゼロ・黒×白×赤）。
- 内容：`content.js`（章とカード）＋ `learnings/*.md`（⑨今日の学びの追記）。

## 章立て（この順）
①超入門 ②開幕・閉店の儀 ③実践工程 ④プロンプト集 ⑤便利コマンド10 ⑥しくじり帳 ⑦詰まったら表 ⑧制作事例 ⑨今日の学び
※ v2で「今日の学び主役」は廃止し、⑨（末尾の追記欄）へ格下げ。③④⑦⑧は骨子のみ（第2段で充実）。

## ⭐ 工程カードを足す・直す（`content.js`）
`content.js` の該当章の `cards: [ ... ]` に**カードオブジェクトを追加**する。フィールド：

```js
{
  no: '05',                 // 工程番号（丸バッジ）。任意
  role: 'マリオ',            // 担当タグ（色ドット付き。ROLES に色を定義）
  title: 'タイトル',
  body: '本文2〜3行',
  point: 'ここで事故る/これだけ先に…（要点コメント・実務の勘所）',
  code: 'コマンド\n複数行OK',  // 黒いコードブロック＋コピーボタン。任意
  prompts: [{ label: '押すと開く見出し', text: '本物のプロンプト全文' }], // 開閉＋コピー。任意
  link: { label: '◯◯を開く', url: 'https://…' },  // 外部リンク。任意
  img: { note: '撮影依頼リスト参照' },              // 画像枠（第2段で流し込み）。任意
}
```
- **プロンプトは HQ実録の本物**（発注文・検収文・引き継ぎ文・修正指示）を入れる。
- 担当タグの色は `content.js` 冒頭の `ROLES` で定義。

## ⭐ 今日の学びを足す（`learnings/NNN.md`）
`learnings/` に次の連番mdを1枚追加→push。⑨の先頭に出る（v1と同じ）。フロントマター：
```
---
title: タイトル
date: 2026-07-21
tags: タグ1, タグ2
---
本文（Markdownサブセット）
```

## 🖼 画像
リポ内に **WebP圧縮**で保存し、カードの `img` から参照（第2段）。
撮影が必要な画面は `撮影依頼リスト.md` にまとめてある（GUI画面は社長がスクショ提供）。

## 🔑 鍵について（簡易鍵の限界）
- 「うっかり見られる」を防ぐ簡易ゲート。**強い秘匿は保証しない**。公開リポのため `content.js`/`learnings/*.md` は鍵なしでも直接読める。
- ゆえに **秘密情報・顧客名・金額・鍵は絶対に載せない**運用で守る。カード追加時は投稿前に目視チェック。
- 将来 Firebase 認証等へ差し替え可能（鍵照合は `index.html` の `KEYHASH` 1箇所）。

## skills との住み分け
`skills/` はAI向けの手順書、**虎の巻は人も読める教科書版**。重複させず、必要なら虎の巻から skill を参照リンクする。
